## 二、发现的问题与修复

### 问题 1：后端日志捕获为空

**现象**：执行蓝图出现后，后端日志文件只有 uvicorn access log，没有 ExecutionEngine 的输出。  
**根因**：Python stdout 默认全缓冲，输出被写入缓冲区但未 flush 到日志文件。  
**修复**：启动命令改为行缓冲模式：

```bash
stdbuf -oL python3 -u main.py
```

验证：修复后重新运行，28 秒内执行蓝图出现，后端日志可实时捕获。

---

### 问题 2：BrowserSkill navigate 卡住

**现象**：Playwright 的 `page.goto()` 使用 `wait_until='networkidle'` 在国内网络下无限等待。  
**根因**：知乎等网站有大量未完成的异步请求，networkidle 条件难以满足。  
**修复**（`blueclaw/skills/browser_skill.py`）：

```python
# 修改前
await page.goto(url, wait_until='networkidle')

# 修改后
await page.goto(url, wait_until='domcontentloaded', timeout=3000)
```

---

### 问题 3：SearchSkill 默认引擎不可访问

**现象**：搜索默认使用 Google，国内环境无法访问导致 Skill 失败。  
**修复**（`blueclaw/skills/search_skill.py`）：

```python
# 修改前
DEFAULT_ENGINE = "google"

# 修改后
DEFAULT_ENGINE = "bing"
```

同时调整结果解析选择器，适配 Bing 搜索结果页 DOM 结构。

---

### 问题 4：ExecutionEngine 硬编码查询词

**现象**：无论用户输入什么，AI 搜索时都在查"杭州3日游"。  
**根因**（`blueclaw/core/execution_engine.py`）：

```python
# _generate_parameters 中的硬编码
query = f"杭州3日游 {step.name} {step.direction}"
```

**修复**：改为直接使用 `step.direction`：

```python
query = step.direction
```

---

### 问题 5：执行超时未完成

**现象**：180 秒等待后执行仍未全部完成，页面持续显示"进行中"。  
**推测根因**：Playwright 实例可能卡在未关闭的浏览器进程上。  
**状态**：BrowserSkill 超时策略已修复（问题 2），但完整执行链路的超时问题未完全验证。

## 四、vis-adapter 面板问题（未修复）

**现状**：后端通过 WebSocket 推送了 base64 截图（`[ExecutionEngine] Screenshot pushed for step xxx`），但前端 `VisualAdapter.tsx` 只显示占位 UI（"拖拽工具到此区域"），不渲染截图。

**根因**：`VisualAdapter.tsx` 没有读取 `useBlueprintStore.screenshots` 数组并渲染的逻辑。

**下一步方案（方案 B：HTML 实时镜像）**：

1. WebAdapter 的 `WebCheckpointManager` 已有 `page.content()` 抓 DOM 能力。
2. 新增 `push_html_snapshot` WebSocket 消息类型，把完整 HTML 传给前端。
3. 前端用 `iframe srcdoc={html}` 渲染，vis-adapter 变成真正的浏览器视图。
4. 需要处理外部资源（CSS/JS/图片）跨域/相对路径断裂问题（URL 重写或 base tag 注入）。

**涉及修改**：
- `blueclaw/core/state_sync.py` — 新增 `push_html_snapshot`
- `blueclaw/adapter/adapters/web.py` — 执行时调用 `page.content()` 并推送
- `frontend/src/components/visual/VisualAdapter.tsx` — iframe 渲染 HTML snapshot
- `frontend/src/components/RealtimeProvider.tsx` — 处理 `html_snapshot` WebSocket 消息
