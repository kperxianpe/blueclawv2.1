### 1.2 模块关系与数据流

#### 关系1：InputScreen → BlueprintCanvas (阶段切换)

```
InputScreen.tsx (用户点击"开始规划")
    │
    ▼ 调用 onStart()
useBlueprintStore.startThinking()
    │
    ▼ phase: 'input' → 'thinking'
    ▼ 调用 generateThinkingNode(0) 生成第一个思考节点
    ▼ thinkingNodes: [firstNode]
    │
    ▼ 状态更新触发 BlueprintCanvas.tsx 重渲染
BlueprintCanvas.tsx
    │
    ▼ 检测到 phase === 'thinking'
    ▼ 渲染左侧 ThinkingBlueprint (ReactFlow画布)
    ▼ 渲染中间 ToolDock (工具栏)
    ▼ 渲染右侧 ExecutionArea + VisualAdapter
```

**关键代码**:
```typescript
// InputScreen.tsx 第16行
<button onClick={() => onStart(input)}>开始规划</button>

// useBlueprintStore.ts 第158-165行
startThinking: () => {
  const firstNode = generateThinkingNode(0);
  set({ phase: 'thinking', thinkingNodes: [firstNode], currentThinkingIndex: 0 });
}

// BlueprintCanvas.tsx 第130-170行 (条件渲染)
{isThinkingVisible && <ThinkingBlueprint ... />}
```

#### 关系2：ThinkingNode → 下一个 ThinkingNode (递进选择)

```
用户点击选项 (如 "A. 自然风光")
    │
    ▼ 调用 selectThinkingOption(nodeId, optionId)
useBlueprintStore.selectThinkingOption()
    │
    ▼ 当前节点 status: 'pending' → 'selected'
    ▼ 记录 selectedOption: 'A'
    │
    ▼ 如果 currentThinkingIndex < 2:
    │   调用 generateThinkingNode(index+1)
    │   新节点添加到 thinkingNodes 数组
    │   currentThinkingIndex++
    │
    ▼ 如果 currentThinkingIndex === 2 (第三轮):
    │   不生成新节点，等待第三个选择完成
    │
    ▼ 状态更新触发 ThinkingBlueprint 重渲染
ReactFlow 画布
    │
    ▼ 新节点自动出现在画布上
    ▼ 节点间通过 Edge 连接（蓝色直线）
    ▼ 新节点可展开继续选择
```

**关键代码**:
```typescript
// useBlueprintStore.ts 第167-188行
selectThinkingOption: (nodeId, optionId) => {
  set((state) => {
    const updatedNodes = state.thinkingNodes.map(n => 
      n.id === nodeId ? { ...n, status: 'selected', selectedOption: optionId } : n
    );
    
    if (state.currentThinkingIndex < 2) {
      const nextNode = generateThinkingNode(state.currentThinkingIndex + 1);
      return { 
        thinkingNodes: [...updatedNodes, nextNode],
        currentThinkingIndex: state.currentThinkingIndex + 1 
      };
    }
    return { thinkingNodes: updatedNodes };
  });
}
```

#### 关系3：ThinkingNode → ExecutionBlueprint (思考转执行)

```
用户完成第三个选择 (点击 "A. 本周末")
    │
    ▼ 调用 selectThinkingOption(thirdNodeId, 'A')
useBlueprintStore.selectThinkingOption()
    │
    ▼ 第三个节点 status: 'pending' → 'selected'
    │
    ▼ 调用 completeThinking()
    ▼   内部调用 generateExecutionBlueprint()
    ▼   生成完整执行蓝图 (6主步骤+3分支+1汇合)
    │
    ▼ phase: 'thinking' → 'execution'
    ▼ executionSteps: [step_001, ..., step_006, branch_01, ..., summary]
    │
    ▼ 状态更新触发 BlueprintCanvas.tsx 重渲染
BlueprintCanvas.tsx
    │
    ▼ 检测到 phase === 'execution'
    ▼ 左侧: ThinkingBlueprint 保持显示 (节点折叠为概览)
    ▼ 右侧上: ExecutionBlueprint 渲染 (ReactFlow画布)
    ▼ 右侧下: VisualAdapter 保持显示
    │
    ▼ 自动调用 executeNextStep() 开始执行
```

**关键代码**:
```typescript
// useBlueprintStore.ts 第190-213行
selectThinkingOption: (nodeId, optionId) => {

        thinkingNodes: updatedNodes,
        executionSteps: executionBlueprint.steps

    }

  });
}

generateExecutionBlueprint() {
  // 6个主路径步骤 + 3个分支 + 1个汇合 + 1个摘要
  return { steps: [...mainSteps, ...branchSteps, convergenceStep, summaryStep] };
}
```

#### 关系4：ExecutionNode → InterferencePanel (节点干涉)

```
用户点击执行节点右上角 ⚙️ 齿轮按钮
    │
    ▼ ExecutionNode 内部状态: showInterference = true
InterferencePanel 组件渲染
    │
    ▼ 显示3个小图标按钮(竖排):
    ▼   ▶️ 蓝色-重新执行
    ▼   🧠 紫色-重新思考
    ▼   ❄️ 青色-冻结执行
    │
用户点击其中一个图标
    │
    ▼ 根据类型调用不同逻辑:
    ▼   'reexecute': 重置当前及后续步骤 → executeNextStep()
    ▼   'rethink':   interveneExecution(stepId) → 回thinking阶段
    ▼   'freeze':    onFreeze(stepName) → BlueprintCanvas设置isFrozen
```

**关键代码**:
```typescript
// ExecutionNode.tsx 第42-77行
const handleInterferenceAction = (action: InterferenceAction) => {
  switch (action) {
    case 'reexecute':
      // 重置后续步骤并重新执行
      const updatedSteps = store.executionSteps.map((s, idx) => 
        idx >= currentIndex ? { ...s, status: 'pending' } : s
      );
      store.executionSteps = updatedSteps;
      setTimeout(() => store.executeNextStep(), 300);
      break;
    case 'rethink':
      store.interveneExecution(stepId); // 回thinking阶段
      break;
    case 'freeze':
      onFreeze?.(step.name); // 触发冻结
      break;
  }
};
```

    ▼ +号按钮 disabled
    │

    │
    ▼ 渲染半透明遮罩 bg-slate-900/10
    ▼ 底部显示按钮栏:
    ▼   📋 更改蓝图形容 (indigo)
    ▼   💖 解释截图 (pink)
