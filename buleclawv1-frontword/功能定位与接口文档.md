## 六、如何扩展接口（后端对接指南）

### 6.1 接入真实AI思考生成

当前 `generateThinkingNode()` 是Mock（固定3轮旅行规划问题），替换为真实AI调用：

```typescript
// src/mock/mockEngine.ts → 改为 src/api/thinkingApi.ts

export async function generateThinkingNode(
  index: number,
  previousSelections?: Selection[],
  customQuestion?: string
): Promise<ThinkingNodeType> {
  // 替换为真实API调用
  const response = await fetch('/api/ai/thinking', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ index, previousSelections, customQuestion })
  });
  return response.json();
}
```

对应修改 `useBlueprintStore.ts` 中的调用点（第159行、第182行）为 `await`。

### 6.2 接入真实执行引擎

当前 `executeNextStep()` 是定时器模拟（2000ms后随机成功/失败），替换为真实执行：

```typescript
// useBlueprintStore.ts 第232-283行
// 替换 executeNextStep 中的模拟逻辑：

// 原代码（模拟）：
// const success = Math.random() > 0.15;
// setTimeout(() => { ... }, 2000);

// 新代码（真实API）：
const response = await fetch(`/api/execute/${step.id}`, {
  method: 'POST',
  body: JSON.stringify({ step, context: store.executionSteps })
});
const result = await response.json();
// 根据 result.status 更新步骤状态
```
